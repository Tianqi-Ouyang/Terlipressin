Code Check (Open)

10. mean() inside rowwise() computes single-row mean

Severity: Moderate File: Tables.Rmd line 871

# Inside rowwise() context:
bili_cal = coalesce(bili_terli_day0, tbili_admit, mean(bili_terli_day0, na.rm = T))

In rowwise() mode, mean(bili_terli_day0) operates on a single value. If bili_terli_day0 is NA, mean(NA, na.rm=T) returns NaN, not the intended column mean. Compute the column mean before the rowwise() pipe.

Variable Definition Check

20. MELD 3.0: sodium not capped at 125–137

Severity: High File: Tables.Rmd lines 923–933, 943–951

# Current (WRONG):
0.82 * (137 - na_terli_day0) - 0.24 * (137 - na_terli_day0) * log(bili_terli_day0)

# Should be (per Kim et al., Gastroenterology 2023):
0.82 * (137 - pmin(pmax(na_terli_day0, 125), 137)) - ...

The published MELD 3.0 formula caps sodium at 125–137 (same as MELD-Na). Uncapped sodium means hyponatremic patients get inflated scores and hypernatremic patients get paradoxically lower scores.

21. MELD 3.0: albumin not capped at 1.5–3.5

Severity: High File: Tables.Rmd lines 923–933, 943–951

# Current (WRONG):
1.85 * (3.5 - alb_terli_day0) - 1.83 * (3.5 - alb_terli_day0) * log(creatinine_adj_day0)

# Should use capped albumin:
alb_capped = pmin(pmax(alb_terli_day0, 1.5), 3.5)
1.85 * (3.5 - alb_capped) - 1.83 * (3.5 - alb_capped) * log(creatinine_adj_day0)

MELD 3.0 caps albumin at 1.5–3.5 g/dL. Patients with albumin > 3.5 get a negative albumin contribution, reducing the score.

22. MELD formulas: bilirubin and INR not floored at 1.0

Severity: Moderate File: Tables.Rmd lines 919, 928–929, 939, 948–949

Per UNOS specification, bilirubin, INR, and creatinine should all be floored at 1.0 (set to 1 if < 1). Creatinine is correctly floored (creatinine_adj_day0), but bilirubin and INR are not. log(x) for x < 1 is negative, reducing the score. Rare in HRS but technically incorrect.

23. CLIF-SOFA kidney failure: threshold should be 2.0, not 1.5

Severity: High File: Tables.Rmd lines 315, 963

# Current (WRONG):
kidney_f = ifelse(!is.na(scr_terli_day0) & scr_terli_day0 >= 1.5, 1, 0)

# Per CLIF-SOFA (Moreau et al., Gastroenterology 2013):
# Kidney failure (CLIF-SOFA = 3) is SCr >= 2.0
# SCr 1.5-1.9 is kidney dysfunction (CLIF-SOFA = 2), NOT failure
kidney_f = ifelse(!is.na(scr_terli_day0) & scr_terli_day0 >= 2.0, 1, 0)

Using ≥ 1.5 overcounts kidney failure, inflating ACLF grades. Directly affects criteria_2 (ACLF ≥ 3) and all futility criteria analyses.

24. CLIF-SOFA circulatory failure: ignores vasopressor requirement

Severity: Moderate File: Tables.Rmd lines 305, 957

# Current:
circulatory_f = ifelse(!is.na(map_terli_day0_time0) & map_terli_day0_time0 < 70, 1, 0)

# Per CLIF-SOFA: circulatory failure (score=3) = use of vasopressors
# MAP < 70 without vasopressors = score 2 (dysfunction, not failure)

The calculate_clif_c_score function correctly uses MAP + vasopressor logic for sub-scoring, but the organ_failure_sum counts MAP < 70 alone as “failure.” Inconsistent between the two functions.

25. AKIN Stage 1: delta uses > 0.3 instead of >= 0.3

Severity: Moderate File: Tables.Rmd line 865

# Current (WRONG):
akin = case_when(
  (ratio >= 1.5 & ratio < 2) | (ratio < 1.5 & delta > 0.3) ~ 1, ...)
                                                    # ^ should be >=

# Correctly done in discont_aki_stage:
  (discont_ratio < 1.5 & discont_delta >= 0.3) ~ 1  # uses >=

AKIN/KDIGO Stage 1 = increase ≥ 0.3 mg/dL. The initial staging uses > while discontinuation staging correctly uses >=. Patients with delta exactly 0.3 are misclassified at initiation.

30. race_2: missing race grouped with White

Severity: Moderate (equity concern) File: Tables.Rmd line 1075

race_2 = if_else(race == 1 | is.na(race), 1, 0) – patients with missing race data are classified as White/Caucasian. Could bias adjusted analyses. Consider a separate “unknown” category.

31. Median imputation for admit_meld_3 and clif_c_score_new

Severity: Moderate (design decision) File: Tables.Rmd lines 1071–1074, 1088

Single-value imputation underestimates variance. These imputed versions are used in criteria flags and potentially in models. The multiple imputation in new_reqeust_0311.Rmd does this more correctly.

32. total_albumin: rowSums(na.rm=TRUE) treats missing days as 0 dose

Severity: Moderate (design decision) File: Tables.Rmd lines 1076–1085

A patient with 7 days of data and 7 missing days gets the same total_albumin as a patient who genuinely received 0g on those 7 days. Systematically underestimates albumin exposure for patients with incomplete data.

34. calculate_clif_c_score parameter named peak_creatinine but receives scr_terli_day0

Severity: Low (naming) File: Tables.Rmd line 211

Misleading parameter name. The function is called with day 0 terlipressin creatinine, not peak hospitalization creatinine.
